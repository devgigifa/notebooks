O que é SQL?

SQL (Structured Query Language) é a linguagem usada pra conversar com bancos de dados relacionais — bancos organizados em tabelas, parecidas com planilhas: linhas e colunas. Com SQL você busca, filtra, junta e modifica dados guardados nessas tabelas.

Pensa assim: se um banco de dados é um arquivo de fichários, cada tabela é uma gaveta (clientes, produtos, pedidos), e cada linha dentro dela é uma ficha individual. SQL é a linguagem que você usa pra pedir "me traga todas as fichas que tenham tal característica".
Sobre este notebook: os comandos aqui rodam num banco SQLite de verdade, carregado no seu navegador. Isso significa que os resultados são reais — não são simulados ou pré-calculados.

BETWEEN e IN

BETWEEN verifica se um valor está dentro de um intervalo (inclusive nas pontas). IN verifica se um valor está numa lista específica de opções.

GROUP BY

GROUP BY agrupa linhas que têm o mesmo valor numa coluna, permitindo aplicar uma função de agregação pra cada grupo separadamente — em vez de pra tabela inteira.

Pensa assim: é como separar um monte de cartas de baralho em pilhas por naipe, e depois contar quantas cartas tem em cada pilha.
group-by.sql
Regra importante: toda coluna no SELECT que não está dentro de uma função de agregação precisa estar também no GROUP BY — como cor no exemplo acima.

INNER JOIN

Até agora consultamos uma tabela por vez. JOIN combina informações de tabelas diferentes, conectando linhas que têm um valor em comum (geralmente uma chave estrangeira). INNER JOIN traz só as linhas que têm correspondência nas duas tabelas.

inner-join.sql
Sobre os apelidos (c, p, ped): quando você junta várias tabelas, é comum dar um apelido curto pra cada uma (com ou sem AS) — isso deixa a query mais curta e evita ambiguidade quando duas tabelas têm coluna com o mesmo nome, como nome aqui.

12

LEFT JOIN

LEFT JOIN traz todas as linhas da tabela da esquerda, mesmo que não tenham correspondência na tabela da direita — nesse caso, as colunas da direita vêm como NULL.

left-join.sql
Quando usar: LEFT JOIN é ótimo pra perguntas do tipo "quais clientes nunca fizeram um pedido?" — algo que INNER JOIN não consegue responder, porque ele descarta linhas sem correspondência.
13

Subqueries

Uma subquery é uma consulta SELECT dentro de outra — usada quando o resultado de uma consulta depende do resultado de outra.

subquery.sql

Essa query traz os produtos com preço acima da média. A subquery entre parênteses roda primeiro (calcula a média), e o resultado dela é usado na condição do WHERE de fora.

14

UNION

UNION combina o resultado de dois SELECTs num só, removendo duplicados automaticamente. As duas consultas precisam ter o mesmo número de colunas.

union.sql
UNION vs UNION ALL: UNION remove duplicados (mais lento). UNION ALL mantém tudo, inclusive repetições (mais rápido). Use UNION ALL quando souber que não vai ter duplicata, ou quando duplicatas não importam.

Chaves primárias e estrangeiras

Chave primária (PRIMARY KEY) identifica cada linha de forma única numa tabela — não pode repetir nem ser nula. Chave estrangeira (FOREIGN KEY) é uma coluna que referencia a chave primária de outra tabela, criando a ligação entre elas.

create-table.sql

Na tabela pedidos deste notebook, cliente_id e produto_id são chaves estrangeiras — cada uma aponta pra um id (chave primária) nas tabelas clientes e produtos.
