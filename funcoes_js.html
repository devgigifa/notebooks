01

O que é uma função?

Uma função é uma caixinha de instruções reutilizável. Você escreve o código uma vez dentro dela, dá um nome pra essa caixinha, e depois é só "chamar" ela sempre que precisar — sem reescrever tudo de novo.

Pensa assim: uma função é como uma receita de bolo. A receita (função) descreve os passos. Você só faz o bolo de verdade (executa) quando decide seguir a receita — e pode fazer isso quantas vezes quiser, até com ingredientes diferentes.

  • function — a palavra que avisa "estou criando uma função"
  • saudacao — o nome que você escolheu pra ela
  • { ... } — as chaves guardam o que a função faz por dentro
Importante: só escrever a função não faz nada acontecer. É preciso chamar ela escrevendo o nome seguido de parênteses: saudacao(). Tente apagar essa linha no editor acima e rodar de novo — o console fica vazio!

02

Parâmetros e argumentos

Uma função fica muito mais útil quando ela pode receber informações de fora. Esses "espaços em branco" que ela recebe se chamam parâmetros. Quando você realmente chama a função e preenche esses espaços, os valores que você passa se chamam argumentos.

Voltando à receita: o parâmetro é tipo "quantidade de açúcar" escrito na receita — um espaço reservado. O argumento é quando você realmente coloca "2 xícaras" na hora de fazer o bolo.

n1 e n2 são os parâmetros — nomes que você escolhe livremente. Eles só existem "dentro" da função, como se fossem gavetas temporárias que se enchem toda vez que a função é chamada.

03

O return

Até agora usamos console.log dentro da função pra mostrar um resultado. Mas na maioria das vezes você quer que a função devolva um valor pra você usar depois, guardar numa variável, ou passar pra outra função. É pra isso que existe o return.

Diferença chave: console.log só imprime algo na tela. return entrega o valor de volta pra quem chamou a função — e assim que o return roda, a função para ali, na hora.

Funções que trabalham com objetos

Objetos em JavaScript guardam informação em pares chave: valor. Uma função pode receber um objeto inteiro como parâmetro e mexer nele — muito comum quando você representa algo do mundo real, tipo um usuário ou um produto.

Arrow functions — a forma curta

Depois que você entende o formato "clássico" de função, vale conhecer a arrow function (função seta), uma forma mais enxuta de escrever a mesma coisa. Ela é muito usada no dia a dia.

arrow.js

As três funções acima fazem exatamente a mesma coisa. A arrow function troca a palavra function por uma seta => e costuma ser guardada numa variável com const. Quando o corpo é uma linha só, dá até pra tirar as chaves e o return fica implícito.
