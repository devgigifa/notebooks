O que é HTML?

HTML significa HyperText Markup Language — linguagem de marcação de hipertexto. Ele não é uma linguagem de programação: não tem lógica, condições ou loops. HTML serve pra uma coisa só: descrever a estrutura de uma página. O que é título, o que é parágrafo, o que é imagem, o que é link.

Pensa assim: se um site fosse uma casa, o HTML é o esqueleto estrutural — paredes, cômodos, portas. O CSS depois vem pintar e decorar, e o JavaScript é o que faz as coisas se mexerem (portas abrindo, luzes acendendo).

HTML é feito de tags — palavras entre <> que marcam onde algo começa e termina:

Estrutura de um documento

<html>Envolve todo o conteúdo da página
<head>Metadados: título da aba, ícone, links de CSS — nada disso aparece na tela
<body>Tudo que realmente é visível na página fica aqui dentro

Div e span

<div> e <span> são "caixas genéricas" sem significado próprio — servem só pra agrupar conteúdo, geralmente pra aplicar estilo com CSS depois. A diferença é o comportamento: div ocupa a linha toda (block); span ocupa só o espaço do conteúdo, ficando na mesma linha do texto ao redor (inline).

HTML semântico

Em vez de usar só <div> pra tudo, HTML moderno tem tags que já dizem o que aquele bloco representa. Isso se chama HTML semântico, e ajuda tanto humanos lendo o código quanto mecanismos de busca e leitores de tela.

semantico.html
Resultado
TagSignificado
<header>Cabeçalho da página ou de uma seção
<nav>Agrupamento de links de navegação
<main>Conteúdo principal da página (só um por página)
<article>Conteúdo autocontido (ex: um post de blog)
<section>Agrupamento temático de conteúdo
<aside>Conteúdo relacionado, mas secundário
<footer>Rodapé da página ou de uma seção
Por que isso importa de verdade: essas tags são fundamentais pra SEO (mecanismos de busca entendem melhor a estrutura da sua página) e acessibilidade (leitores de tela conseguem navegar "pulando" direto pra <nav> ou <main>, por exemplo).
